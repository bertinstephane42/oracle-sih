Comprendre les Tablespaces et Fichiers de Données
Concept clé : Un tablespace est une unité logique de stockage qui contient des objets de base de données (tables, index, etc.), tandis qu'un fichier de données est le fichier physique qui stocke concrètement les données.

◆ Les composants principaux
TABLESPACE — Unité logique de stockage Oracle. Un tablespace regroupe des structures de stockage logiques. C'est la couche intermédiaire entre la base de données et les fichiers de données.
FICHIER DE DONNÉES (Data File) — Fichier physique sur le disque du système d'exploitation. Chaque fichier de données est associé à un seul tablespace, mais un tablespace peut contenir plusieurs fichiers de données.
SEGMENT — Ensemble d'extents alloués à un objet de base de données spécifique (table, index, etc.). Un segment appartient à un seul tablespace mais peut s'étendre sur plusieurs fichiers de données.

◆ Détail de chaque niveau

🏚 Database (Base de données)

Ensemble complet des données et structures Oracle. C'est le niveau le plus élevé qui contient tous les tablespaces.

📦 Tablespace

Unité logique de stockage qui regroupe des structures de stockage. Un tablespace :

  • Contient un ou plusieurs fichiers de données
  • Peut contenir plusieurs segments (tables, index)
  • Est l'unité de sauvegarde et de récupération
  • Peut être en ligne ou hors ligne

🗃 Data File (Fichier de données)

Fichier physique sur le disque du système d'exploitation. Caractéristiques :

  • Appartient à un seul tablespace
  • Contient les données réelles
  • Peut être redimensionné (auto-extensible)
  • Stocké sur le système de fichiers ou ASM

🗃️ Segment

Ensemble d'extents alloués à un objet spécifique. Types de segments :

  • Table segment : contient les données d'une table
  • Index segment : contient les données d'un index
  • Undo segment : contient les données d'annulation
  • Temporary segment : pour les opérations temporaires

📐 Extent

Ensemble contigu de blocs de données Oracle. Un extent :

  • Est alloué à un segment spécifique
  • Contient plusieurs blocs de données
  • Est alloué quand un segment grandit
  • Ne peut pas être partagé entre segments

🔹 Data Block (Bloc de données)

Plus petite unité de stockage Oracle. Le bloc :

  • Est la plus petite unité d'E/S Oracle
  • Taille typique : 8 Ko (configurable)
  • Contient des en-têtes et des données
  • Correspond à une ou plusieurs blocs OS
◆ Guide de bonnes pratiques — Création d'une base Oracle
Principe : Séparez vos tables et index dans des tablespaces distincts, et répartissez les fichiers de données sur plusieurs disques physiques pour optimiser les E/S concurrentes.
  1. Créer les tablespaces cibles — un pour les données, un pour les index
-- Tablespace pour les données (3 data files sur disques distincts)
CREATE TABLESPACE tbs_data
  DATAFILE
   '/u01/oradata/prod/tbs_data01.dbf' SIZE 10G AUTOEXTEND ON,
   '/u02/oradata/prod/tbs_data02.dbf' SIZE 10G AUTOEXTEND ON,
   '/u03/oradata/prod/tbs_data03.dbf' SIZE 10G AUTOEXTEND ON
  EXTENT MANAGEMENT LOCAL UNIFORM SIZE 1M
  SEGMENT SPACE MANAGEMENT AUTO;

-- Tablespace pour les index
CREATE TABLESPACE tbs_index
  DATAFILE '/u04/oradata/prod/tbs_index01.dbf' SIZE 5G AUTOEXTEND ON
  EXTENT MANAGEMENT LOCAL UNIFORM SIZE 128K
  SEGMENT SPACE MANAGEMENT AUTO;
  1. Assigner les objets aux bons tablespaces à la création
-- Table dans tbs_data
CREATE TABLE hr.employees (
  emp_id  NUMBER(6) PRIMARY KEY,
  emp_name VARCHAR2(100)
) TABLESPACE tbs_data;

-- Index dans tbs_index (séparation physique)
CREATE INDEX hr.emp_name_idx ON hr.employees(emp_name)
  TABLESPACE tbs_index;
Résultat : Les lectures/index s'exécutent en parallèle sur des disques distincts, réduisant la contention d'E/S et améliorant les temps de réponse jusqu'à 30–50 % sur les requêtes mixtes (lectures + index).

◆ Comment ça marche ?
Processus d'allocation
  1. Quand vous créez une table, Oracle alloue un segment dans le tablespace
  2. Le segment commence avec un extent initial (premier ensemble de blocs)
  3. Quand l'extent est plein, Oracle alloue un nouvel extent
  4. Les extents peuvent être dans le même fichier ou dans d'autres fichiers du tablespace
  5. Chaque extent contient plusieurs blocs de données (typiquement 8 Ko chacun)
Fragmentation : Avec le temps, les suppressions et insertions peuvent créer de la fragmentation. Oracle utilise différents mécanismes (comme ASSM — Automatic Segment Space Management) pour optimiser l'utilisation de l'espace.

Glossaire complet
Cliquez sur chaque terme pour afficher sa définition

📊 Tablespace

Unité logique de stockage dans Oracle qui contient des objets de base de données. Un tablespace est composé d'un ou plusieurs fichiers de données physiques.

Caractéristiques :

  • Peut être en ligne (ONLINE) ou hors ligne (OFFLINE)
  • Peut être en lecture seule (READ ONLY) ou lecture-écriture (READ WRITE)
  • Peut être temporaire (TEMPORARY) ou permanent (PERMANENT)
  • Supporte l'auto-extension des fichiers de données

🗃️ Data File (Fichier de données)

Fichier physique sur le disque du système d'exploitation qui stocke les données d'un tablespace Oracle.

Caractéristiques :

  • Appartient exclusivement à un seul tablespace
  • Peut être redimensionné manuellement ou automatiquement
  • Stocké sur système de fichiers ou ASM (Automatic Storage Management)
  • Extension typique : .dbf

📦 Segment

Structure de stockage qui contient les données d'un objet de base de données spécifique (table, index, etc.).

Types de segments :

  • Table segment : Stocke les lignes d'une table
  • Index segment : Stocke les entrées d'index
  • Undo segment : Stocke les informations d'annulation
  • Temporary segment : Utilisé pour les opérations de tri temporaires
  • LOB segment : Stocke les objets Large Object (BLOB, CLOB)

📐 Extent

Ensemble contigu de blocs de données Oracle alloué à un segment.

Caractéristiques :

  • Unité d'allocation d'espace pour les segments
  • Taille définie lors de la création du segment (INITIAL EXTENT)
  • Peut augmenter selon NEXT EXTENT et les paramètres d'extension
  • Ne peut pas être partagé entre plusieurs segments

🔹 Data Block (Bloc de données)

Plus petite unité de stockage et d'E/S dans Oracle.

Structure d'un bloc :

  • En-tête (Header) : Informations générales sur le bloc
  • Table des lignes (Row Directory) : Pointeurs vers les lignes
  • Espace libre (Free Space) : Espace disponible pour nouvelles données
  • Données (Row Data) : Les données réelles stockées

Taille configurable : 2K, 4K, 8K (défaut), 16K, 32K

🔄 ASM (Automatic Storage Management)

Système de gestion de stockage intégré à Oracle qui simplifie l'administration des fichiers de données.

Avantages :

  • Équilibrage automatique des E/S (striping)
  • Miroir intégré pour la redondance
  • Gestion simplifiée des fichiers de données
  • Redimensionnement dynamique des disques

⚡ ASSM (Automatic Segment Space Management)

Mécanisme Oracle pour gérer automatiquement l'espace libre dans les segments.

Fonctionnement :

  • Utilise des bitmaps au lieu des freelists
  • Gère automatiquement l'espace libre
  • Réduit la fragmentation
  • Améliore les performances en environnement concurrentiel

📝 UNDO Tablespace

Tablespace spécialisé qui stocke les données d'annulation pour les transactions.

Utilisations :

  • Rollback des transactions non validées
  • Lecture cohérente (consistent read)
  • Récupération après échec (failure recovery)
  • Flashback Query

🗃️ TEMP Tablespace

Tablespace temporaire utilisé pour les opérations de tri et les tables temporaires.

Utilisations typiques :

  • Opérations ORDER BY et GROUP BY
  • Jointures complexes (hash joins, sort merges)
  • Création d'index
  • Tables temporaires globales (GTT)

Lexique technique — Architecture Oracle
Concepts de stockage
  • Tablespace — Unité logique de stockage regroupant des structures de base de données (tables, index), composée d'un ou plusieurs fichiers de données physiques.
  • Fichier de données (Data File) — Fichier physique sur le système d'exploitation stockant les données d'un tablespace. Extension typique : .dbf.
  • Segment — Ensemble d'extents alloué à un objet spécifique (table, index, undo, temporaire, LOB).
  • Extent — Ensemble contigu de blocs de données alloué à un segment. Unité d'allocation d'espace.
  • Bloc de données (Data Block) — Plus petite unité d'E/S Oracle. Taille par défaut : 8 Ko.
Gestion de l'espace
  • ASM (Automatic Storage Management) — Système de stockage intégré Oracle offrant striping, miroir et gestion simplifiée des fichiers.
  • ASSM (Automatic Segment Space Management) — Gestion automatique de l'espace libre via bitmaps, remplaçant les freelists.
  • Tablespace SYSTEM — Tablespace système contenant le dictionnaire de données Oracle.
  • Tablespace SYSAUX — Tablespace auxiliaire réduisant la charge sur SYSTEM pour les composants optionnels.
  • Tablespace UNDO — Stocke les données d'annulation pour rollback, read consistency et flashback query.
  • Tablespace TEMP — Espace temporaire pour les tris, jointures et tables temporaires globales (GTT).
  • Tablespace USERS — Tablespace par défaut pour les objets des utilisateurs non système.
