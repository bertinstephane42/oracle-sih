Partitionnement Avance — Algorithme de Pruning
Principe

Le Partition Pruning est un mecanisme interne du CBO qui elimine les partitions non necessaires a la resolution d'un predicat. Avec un partitionnement Range par mois sur 120 partitions (10 ans), une requete filtrant sur les 6 derniers mois ne scanne que 6 partitions au lieu de 120.

Complexite Algorithmique

ScanTime(rows) = rows / (ioBandwidth × pruningFactor)

Avec pruningFactor = totalPartitions / scannedPartitions. Sans partitionnement, pruningFactor = 1. Avec pruning sur 1 mois sur 120, pruningFactor = 120.

  • Metadonnees de partition : Oracle consulte les HIGH_VALUE/LOW_VALUE de chaque partition (stockes dans USER_TAB_PARTITIONS) pour elaguer avant tout acces.
  • Partition Exchange Loading (PEL) : Permet d'echanger une partition avec une table Oracle compatible via une operation essentiellement metadata only. Cette technique est critique pour le chargement massif et l'archivage sans impact significatif sur l'OLTP.
  • Maintenance : TRUNCATE PARTITION et EXCHANGE PARTITION sont essentiellement des operations metadata only. En revanche, MERGE PARTITION et SPLIT PARTITION peuvent necessiter des mouvements ou reecritures de donnees selon la structure physique et les index locaux/globaux.
Contexte HDS / COFRAC

La conservation des historiques de prescriptions sur 10+ ans est requise par la reglementation COFRAC. Le partitionnement permet de conserver l'acces aux donnees archivees sans impacter les requetes OLTP courantes. Le PEL permet d'archiver les partitions les plus anciennes sur un stockage a froid (TABLESPACE_ARCHIVE) tout en preservant la capacite de requetage.

In-Memory Column Store — IMCU & Scan SIMD
Principe

L'option Oracle Database In-Memory stocke une copie des donnees au format colonne (Column Store) dans l'In-Memory Area de la SGA, en plus du format ligne (Row Store) conserve dans le Buffer Cache. Les IMCUs (In-Memory Compression Units) sont des unites de stockage columnaires comprimees, associees a des SMU (Storage Metadata Units) contenant les metadonnees de pruning et de scan.

Architecture IMCU
  • Transposition Row→Column : Les colonnes selectionnees sont extraites des lignes et stockees consecutivement en memoire. Une requete SELECT SUM(qty) ne lit que les blocs colonne de qty, pas les 199 autres colonnes.
  • Compression IMCU : Chaque IMCU est comprime avec un dictionnaire local (FOR ALL COLUMNS ou MEMCOMPRESS). Ratio typique 8x a 15x pour les donnees alphanumeriques.
  • Scan Vectorise SIMD : Les CPU modernes (AVX2 256-bit) traitent 8 valeurs entieres par instruction (Single Instruction Multiple Data). Oracle vectorise les scans colonne pour atteindre des debits de 10-15 Go/s par socket.
  • Predicate Pushdown et IMCU Pruning : Les predicats WHERE sont appliques directement lors du scan colonne (ex: WHERE service = 'CARDIO'). Oracle elimine les IMCUs non pertinentes via les metadonnees de min/max stockees dans les SMU, reduisant fortement le volume scanne.
Formules

ScanTime = (rows × rowWidth × colSelectivity) / (imcuBandwidth × vectorFactor)

EffectiveScanReduction = selectedCols / totalCols

CompressionRatio ≈ originalSize / compressedIMCUSize

Avec vectorFactor = 6 (estimation conservative du gain SIMD sur 256-bit AVX2), imcuBandwidth = 12 GB/s (bande passante memoire DDR4-3200 dual channel par socket).

OCI Data Flow — Spark & Architecture Polyglotte
Principe

OCI Data Flow est un service Apache Spark managé et serverless sur Oracle Cloud. Le service provisionne automatiquement un cluster Spark ephemere pour l'execution des jobs, sans administration manuelle d'infrastructure par l'utilisateur. L'architecture est polyglotte : Oracle SQL pour le transactionnel et l'analytique structure, Spark pour les traitements massifs et non structurés (logs, genomique, imagerie).

Architecture Distribuee
  • Spark Executor Memory : Chaque nœud executer alloue une portion de RAM (spark.executor.memory) pour les operations en memoire. Les donnees depassant cette capacite sont spillées sur disque (identique au mécanisme PGA Temp Spill).
  • Shuffle Partitions : Lors d'un GROUP BY ou JOIN, Spark repartitionne les donnees entre les executors (shuffle). Le parametre spark.sql.shuffle.partitions controle le parallelisme. Defaut : 200 partitions.
  • Predicate Pushdown Oracle : Le connecteur Oracle Database pour Spark (JDBC / Oracle datasource) permet de pousser les filtres WHERE dans Oracle (via Oracle SQL), reduisant le volume de donnees transferees sur le reseau. Seules les lignes correspondant aux predicats sont lues.
  • Data Locality : Spark tente de programmer les tasks sur les nœuds où les donnees resident. En OCI, la localite est optimale quand les donnees Oracle sont dans un Autonomous Database sur la meme region (eu-france-1).
Contexte HDS / Génomique

Le sequencage genomique produit des fichiers FASTQ de 50-200 Go par patient. Leur analyse (alignement, variant calling) depasse la capacite d'une base Oracle unique sans cout prohibitif. L'architecture polyglotte (Oracle pour le master data patient + Spark pour le calcul distribué) est la seule approche viable certifiable HDS. OCI Data Flow s'execute dans le perimetre OCI France certifié HDS, ISO 27001. Aucune donnee de santé ne quitte la région eu-france-1.

Un score > 90% indique un passage réussi du Big Data au Fast Data pour les 5 milliards de lignes.

Lexique technique — Advanced Performance SIH
Architecture Oracle Fondamentale
  • Conteneur de Base de Donnees (CDB) — Base de donnees racine chapeautant l'ensemble des bases de donnees placables dans l'architecture multitenant Oracle 12c et suivantes.
  • Base de Donnees Placable (PDB) — Base de donnees portable et isolee au sein d'un conteneur, apparaissant aux utilisateurs et aux applications comme une base de donnees classique.
  • Dictionnaire Partage — Ensemble de metadonnees de definition (structures de schemas, objets, privileges) mutualisees entre toutes les bases placables d'un meme conteneur.
  • Cache de Lignes du Dictionnaire — Structure memoire logeant dans la memoire partagee les lignes du dictionnaire, protegee par un verrou d'exclusion mutuelle (latch).
  • Masquage de Donnees — Technique d'obfuscation des donnees sensibles (numero d'inscription au repertoire, identite des patients) lors du clonage d'une base, conforme au Reglement General sur la Protection des Donnees.
Memoire Globale (SGA / PGA)
  • Memoire Globale de Systeme (SGA) — Zone memoire partagee par l'ensemble des processus Oracle, abritant le cache de tampons, le reservoir partage, le tampon de journalisation et les structures connexes.
  • Cache de Tampons — Portion de la memoire globale contenant les blocs de donnees lus depuis les fichiers disques, ordonnancee par l'algorithme du moindre recent usage.
  • Memoire Globale de Programme (PGA) — Zone memoire privee allouee a chaque session d'utilisation pour les operations de tri, de regroupement et de jointure.
  • Debordement Temporaire — Basculement des operations memoire vers les espaces temporaires sur disque lorsque la memoire globale de programme est insuffisante, entrainant une degradation sensible des temps d'execution.
  • Compteur d'Acces (Touch Count) — Indicateur du nombre de consultations d'un bloc dans le cache de tampons, determinant son placement dans la chaine du moindre recent usage.
  • Verrou d'Exclusion Mutuelle (Latch) — Mecanisme de verrouillage leger protegeant l'acces concurrent aux structures partagees de la memoire globale.
Transactions et Coherence
  • Numero de Changement Systeme (SCN) — Compteur sequentiel global horodatant chaque transaction et garantissant l'ordre causal des modifications dans l'ensemble de la base.
  • Journal de Retablissement (Redo) — Enregistrement sequentiel de toutes les modifications appliquees a la base, utilise pour la reprise apres incident par application differee.
  • Segment d'Annulation (Undo) — Blocs conservant l'image anterieure des donnees modifiees, necessaires a la lecture coherente et au retour en arriere des transactions.
  • Reprise apres Incident (Instance Recovery) — Processus automatique de retablissement de la base apres un arret brutal, combinant l'application du journal de retablissement (avance rapide) et l'application des segments d'annulation (retour arriere).
  • Coherence de Lecture — Mecanisme garantissant qu'une requete percoit un instantane coherent des donnees a un instant de reference, en utilisant les blocs d'annulation.
  • Erreur d'Instantané Trop Ancien (ORA-01555) — Erreur survenant lorsque les blocs d'annulation necessaires a la lecture coherente d'une longue requete ont ete ecrases par des transactions plus recentes avant la fin de la consultation.
Optimiseur et Plans d'Execution
  • Optimiseur Base sur les Couts (CBO) — Moteur d'optimisation d'Oracle analysant les statistiques des objets et les couts d'acces pour determiner le plan d'execution le plus efficace pour chaque requete.
  • Plan d'Execution — Arbre d'operations (acces tables, jointures, tris) genere par l'optimiseur pour executer une requete SQL, visible via la commande EXPLAIN PLAN.
  • Parcours Integral de Table (Full Table Scan) — Methode d'acces lisant l'integralite des blocs d'une table, utilisee en l'absence d'index adapte ou lorsque la table est de petite taille.
  • Parcours Unique par Index (Index Unique Scan) — Methode d'acces retournant au plus une ligne via un index unique, typique des recherches par cle primaire.
  • Jointure par Hash (HASH JOIN) / Boucles Imbriquees (NESTED LOOPS) — Algorithmes de jointure : le premier convient aux grands volumes sans index, le second aux petites tables indexees.
PL/SQL et Programmation
  • Chargement Massif en Collection (BULK COLLECT) — Directive PL/SQL chargeant un ensemble de lignes en une seule operation dans une collection memoire (PGA), reduisant les allees-retours entre les moteurs SQL et PL/SQL.
  • Mise a Jour Massive (FORALL) — Instruction PL/SQL executant une operation DML sur toutes les lignes d'une collection sans commutation de contexte individuelle, accelerant les traitements par lots.
  • Commutation de Contexte (Context Switch) — Basculement entre le moteur SQL et le moteur PL/SQL, dont le cout cumule peut etre significatif dans les boucles iteratives.
  • Declencheur (Trigger) — Bloc PL/SQL stocke s'executant automatiquement sur un evenement de modification (INSERT, UPDATE, DELETE) d'une table, utilise pour les regles metier et les contraintes complexes.
  • Collection PL/SQL — Structure de donnees programmee (table imbriquee, VARRAY, table associative) permettant de manipuler des ensembles de lignes en memoire.
Securite et Gouvernance HDS
  • Chiffrement Transparent (TDE) — Mecanisme de chiffrement des donnees au repos au sein des fichiers et des espaces de tables, sans aucune modification des applications clientes, utilisant l'algorithme AES a clef de 256 bits.
  • Moindre Privilege (Least Privilege) — Principe de securite selon lequel un utilisateur ne dispose que des droits strictement necessaires a ses fonctions, reduisant la surface d'attaque et les risques d'erreur.
  • Masquage Dynamique (DBMS_REDACT) — Paquetage Oracle permettant le masquage a la volee des donnees sensibles dans les resultats de requetes, sans alterer les donnees stockees.
  • Journal d'Activite (Audit HDS) — Enregistrement chiffre et horodate de l'ensemble des actions effectuees sur le systeme, garantissant la tracabilite requise par la certification Hebergement de Donnees de Sante.
Partitionnement Avance
  • Elagage de Partitions (Partition Pruning) — Mecanisme interne de l'optimiseur (CBO) qui elimine du plan d'execution les partitions ne repondant pas aux predicats d'une clause de filtrage, reduisant proportionnellement le volume d'entrees-sorties et le temps de reponse.
  • Echange par Chargement de Partition (PEL) — Operation metadier de chargement consistant a echanger une partition existante avec une table externe par simple mise a jour du dictionnaire de donnees, evitent la reconstruction des index et la journalisation ligne a ligne.
  • Partitionnement par Intervalles (Interval Partitioning) — Extension du partitionnement par plage (Range) dans laquelle Oracle cree automatiquement de nouvelles partitions lors de l'insertion de donnees depassant la limite haute, sans intervention manuelle de l'administrateur.
  • Facteur d'Elagage (Pruning Factor) — Rapport entre le nombre total de partitions et le nombre de partitions effectivement parcourues apres application des predicats, mesurant l'efficacite de la reduction du volume d'entrees-sorties par l'elagage.
  • Hierarchisation du Stockage (Storage Tiering) — Strategie de placement des partitions sur des supports physiques de performances differentes (NVMe, SSD, HDD, stockage froid) selon leur age et leur frequence d'acces, combinee a l'archivage des partitions anciennes.
Stockage Colonnaire et Acceleration Vectorielle
  • Unite de Compression Colonnaire en Memoire (IMCU) — Unite de stockage interne a la zone In-Memory de la SGA contenant les donnees d'un segment au format colonne, compressees et organisees en blocs de taille fixe pour un acces vectoriel optimise.
  • Unite de Metadonnees de Stockage (SMU) — Structure de gestion associee a chaque IMCU, contenant les metadonnees d'elagage, les dictionnaires de codage par colonne et les informations necessaires aux acces concurrents et a la coherence transactionnelle.
  • Entrepot au Format Colonnaire (Column Store) — Mode d'organisation des donnees par colonnes plutot que par lignes, permettant de ne lire que les colonnes referencees par la requete, ce qui reduit le volume d'entrees-sorties et maximise l'efficacite de la compression.
  • Parallelisme Vectoriel (SIMD) — Technique de parallelisme au niveau du processeur appliquant une meme operation arithmetique ou logique a plusieurs valeurs simultanement via des registres vectoriels larges (256 bits avec AVX2, 512 bits avec AVX-512).
  • Facteur d'Acceleration Vectorielle — Rapport de performance estime entre l'execution scalaire classique et l'execution vectorielle SIMD, fonction de la largeur des registres du processeur et de la bande passante memoire disponible.
Traitement Distribue et Architecture Polyglotte
  • Apache Spark — Moteur de traitement unifie pour l'analyse de donnees volumineuses, fonde sur le calcul distribue en memoire (RDD / DataFrame) et l'execution parallelisee sur un ensemble de noeuds de travail ephemeres.
  • Architecture Polyglotte — Approche combinant plusieurs systemes de gestion de donnees aux modeles differents au sein d'une meme infrastructure, ou Oracle Database assume le transactionnel structure et Spark les traitements massifs non structures (logs, genomique, imagerie).
  • Calcul Sans Serveur (Serverless) — Modele d'execution ou le fournisseur de cloud provisionne et dimensionne automatiquement l'infrastructure de calcul, liberant l'utilisateur de la gestion des serveurs et de la planification de capacite.
  • Processeur d'Execution Distribue (Executor) — Processus de travail lance sur un noeud d'un cluster Spark, responsable de l'execution des tâches sur une partition des donnees, dote d'une memoire allouee et d'un nombre defini de coeurs de calcul.
  • Desequilibre des Donnees (Data Skew) — Distribution inegale des donnees entre les partitions d'un systeme distribue, provoquant la sous-utilisation de certains noeuds et la saturation d'autres par effet de goulet d'etranglement.
