Objectif

Cet atelier vous propose 18 scénarios métiers tirés du contexte d'une Pharmacie à Usage Intérieur (PUI). Pour chaque défi, vous devez rédiger la requête SQL ou le bloc PL/SQL qui résout le problème posé. Un moteur de validation syntaxique et logique vous assiste dans votre apprentissage.

  • Sélection : Choisissez un scénario dans le menu déroulant de la barre latérale.
  • Éditeur : Rédigez votre solution dans la zone de texte dédiée.
  • Vérification : Le bouton « Vérifier la syntaxe » analyse votre code et signale les erreurs courantes.
  • Exécution : Lancez votre requête pour voir le résultat simulé.
  • Solution Expert : Consultez la solution optimale commentée ligne par ligne.

Lexique technique — Oracle SQL Helper
Concepts fondamentaux
  • Jointure (JOIN) — Opération relationnelle combinant les lignes de deux tables selon une condition de correspondance entre leurs colonnes. La jointure interne (INNER JOIN) ne retient que les paires vérifiant la condition ; la jointure externe (LEFT/RIGHT/FULL OUTER JOIN) conserve également les lignes orphelines de la table préservée.
  • Agrégation — Fonction de synthèse (SUM, AVG, COUNT, MIN, MAX) réduisant un ensemble de lignes en une valeur unique. La clause GROUP BY partitionne les lignes en groupes avant agrégation ; la clause HAVING filtre les groupes après agrégation.
  • Sous-requête Corrélée — Requête imbriquée dont la condition interne fait référence à une colonne de la requête externe, provoquant sa réévaluation pour chaque ligne de l'ensemble parent. Alternative : fonction analytique ou jointure, souvent plus performantes.
  • BULK COLLECT — Directive PL/SQL chargeant un ensemble complet de lignes dans une collection mémoire en une seule transition entre le moteur SQL et le moteur PL/SQL, réduisant drastiquement le nombre de changements de contexte.
  • Déclencheur (Trigger) — Bloc PL/SQL stocké, exécuté automatiquement sur un événement DML (INSERT, UPDATE, DELETE) d'une table. Utilisé pour l'audit, les règles métier et les contraintes complexes.
  • Vue (View) — Table virtuelle issue d'une requête SELECT stockée comme objet du schéma. Ne contient aucune donnée propre mais offre abstraction, masquage et simplification des requêtes.
  • Vue Matérialisée (Materialized View) — Vue dont les résultats sont physiquement stockés sur disque, actualisables par REFRESH (COMPLETE, FAST, ON DEMAND, ON COMMIT). Optimale pour les rapports pré-calculés.
  • Index Unique — Index B-tree garantissant l'unicité des valeurs d'une colonne ou d'une combinaison de colonnes. Créé automatiquement par les contraintes PRIMARY KEY et UNIQUE.
  • Partitionnement — Division d'une table volumineuse en segments physiques indépendants selon une clé de partition (Range, List, Hash). L'élagage (Partition Pruning) permet à l'optimiseur de ne consulter que les segments pertinents.
  • Séquence (Sequence) — Générateur de nombres uniques et incrémentaux, utilisé pour les clés primaires. NEXTVAL retourne la valeur suivante ; CURRVAL la dernière valeur générée dans la session courante.
  • Gestion d'Exceptions — Mécanisme PL/SQL de capture et traitement des erreurs via les blocs EXCEPTION. Exceptions prédéfinies : NO_DATA_FOUND, TOO_MANY_ROWS, ZERO_DIVIDE.
  • Curseur Explicite — Pointeur déclaré (CURSOR c IS SELECT...) permettant le parcours ligne à ligne d'un ensemble de résultats, avec ouverture, extraction et fermeture contrôlées par le programmeur.
  • MERGE (Upsert) — Instruction conditionnelle combinant INSERT et UPDATE en une seule passe : si la ligne cible existe, elle est mise à jour ; sinon, elle est insérée. Syntaxe : MERGE INTO cible USING source ON (condition) WHEN MATCHED THEN UPDATE WHEN NOT MATCHED THEN INSERT.
  • Fonction Analytique — Fonction de fenêtrage (ROW_NUMBER(), RANK(), DENSE_RANK(), LAG(), LEAD()) calculant une valeur sur un ensemble de lignes défini par la clause OVER (PARTITION BY ... ORDER BY ...).
  • CTE (WITH clause) — Expression de table commune définissant un jeu de résultats temporaire nommé, utilisable dans la requête principale et récursivement pour les structures hiérarchiques.
  • Verrouillage (LOCK) — Mécanisme de contrôle de concurrence. SELECT ... FOR UPDATE verrouille les lignes sélectionnées, empêchant toute modification concurrente jusqu'à la fin de la transaction. NOWAIT et SKIP LOCKED modifient le comportement d'attente.
  • SQL Dynamique (EXECUTE IMMEDIATE) — Instruction PL/SQL construisant et exécutant une commande SQL à la volée, dont la structure (table, colonne, condition) peut être déterminée à l'exécution plutôt qu'à la compilation.
