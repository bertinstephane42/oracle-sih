★ Etape 1 — Installation & Architecture
Fondations du projet PUI. Choix du Character Set et configuration du Resource Plan CDB/PDB. Ces decisions sont irreversibles apres creation de la base. Impact direct sur l'internationalisation et l'isolation des ressources.
A WE8MSWIN1252 + Resource Plan par defaut Encodage 8-bit
B AL32UTF8 + Resource Plan CDB avec shares PUI dedies Encodage Unicode
C AL32UTF8 mais SANS Resource Plan actif Sans Resource Plan
🔒 Etape 2 — Securite & Conformite HDS/RGPD
Configuration du chiffrement TDE, de l'audit HDS et des profils de securite. En environnement hospitalier, la non-conformite expose a des sanctions juridiques et a la suspension de l'habilitation HDS.
A Pas de TDE, Audit desactive (gain perf estime 5%) Audit desactive
B TDE AES-256 + Audit HDS standard + Profil SECURE_PUI Audit complet
C Audit partiel (echecs seulement), TDE sur TS DATA seulement Audit partiel
💻 Etape 3 — Developpement & Requetage
Revue de code PL/SQL avant mise en production. Les choix d'ecriture SQL impactent directement les performances de la PUI et la charge sur le serveur de base de donnees.
A SELECT * FROM stock_med s JOIN ligne_disp l ON s.med_id = l.med_id Colonnes implicites
B SELECT s.med_id, s.molecule, l.qte FROM stock_med s JOIN ligne_disp l ON s.med_id = l.med_id Colonnes explicites
C Curseur PL/SQL ligne a ligne sans BULK COLLECT Parcours ligne/ligne
⚙ Etape 4 — Tuning Systeme (SGA/PGA/Undo)
Ajustement des parametres memoire face a une charge simulee de reporting PMSI/T2A. Un mauvais dimensionnement entraine des ORA-01555 et des temps de reponse degradés.
A UNDO_RETENTION=900s, SGA=2G, pas de tuning UNDO par defaut
B UNDO_RETENTION=1800s, SGA=4G, PGA=1G, undo tablespace 4G UNDO dimensionne
C SGA=8G mais UNDO_RETENTION=600s, pas de monitoring SGA sur-dimensionnee
📊 Etape 5 — Tests de Charge & Recette
Simulation de 500 utilisateurs concurrents. Un ralentissement est detecte. A vous d'identifier la cause a travers les Wait Events et de proposer la correction.
A Ignorer les Wait Events, augmenter la memoire (solution palliative) Correction memoire
B Analyser enq: TX - row lock contention, corriger le CACHE de sequence Diagnostic Wait Event
C Rebuild de tous les index (couteux, sans diagnostic) Rebuild index
🌎 Etape 6 — Mise en Production & Maintenance
Validation de la strategie de Backup/PRA avant le Go-Live. La capacite a restaurer les donnees est la seule verite en cas de sinistre.
A Valider sans test de restauration, backup journalier seulement Backup simple
B RMAN validate + Data Guard Physical Standby + test trimestriel Backup + Standby
C Export logique expdp seulement (pas de RMAN ni standby) Export logique

Guide de Deploiement — Vue d'Ensemble
Ce serious game vous guide a travers les 6 etapes du cycle de vie d'un projet Oracle en contexte hospitalier PUI :

Etape 1 — Installation : Choix du Character Set (AL32UTF8 vs legacy), Resource Plan CDB/PDB
Etape 2 — Securite : TDE, Audit HDS, Profils de securite — conformite obligatoire
Etape 3 — Developpement : Revue de code PL/SQL, SELECT explicite, indexation
Etape 4 — Tuning : SGA/PGA/Undo — eviter ORA-01555 et les temps de reponse degradés
Etape 5 — Recette : Analyse des Wait Events sous charge, contention de sequence
Etape 6 — Production : Backup RMAN, Data Guard, PRA — la restauration est la seule verite

Chaque choix a des consequences visibles immediatement. Utilisez le bouton "Voir la Bonne Pratique" si vous souhaitez de l'aide.
